# QUICKSTART

Getting basictexts.org running locally takes about 2 minutes.

## Prerequisites

- [Node.js](https://nodejs.org/) v18 or later (v20+ recommended)
- npm v9+

## Clone and install

```bash
git clone https://github.com/idusortus/recovery-basic-texts.git
cd recovery-basic-texts
npm install
```

## Build the search index

The search index is a prebuilt static asset generated from the corpus files.
It is not checked into the repository (it's in `.gitignore`), so you must build it before running the app:

```bash
npm run build:index
```

This reads `corpus/sources.json` and all `corpus/sources/*.json` files and emits:
- `static/index/minisearch.json` — the serialized search index
- `static/index/passages.json` — passage lookup map
- `static/index/index-meta.json` — version metadata

## Start the dev server

```bash
npm run dev
```

Open [http://localhost:5173](http://localhost:5173).

The search bar is on the home page. Try searching for: `fear`, `powerless`, `solution`,
or `"how it works"` (quoted phrase).

## Validate the corpus

```bash
node corpus/scripts/validate.js
```

Checks all enabled corpus files for schema correctness and referential integrity.

## Full production build

```bash
npm run build
```

This runs `build:index` followed by `vite build` using the Cloudflare Pages adapter.
Output goes to `.svelte-kit/cloudflare/`.

## TypeScript / Svelte check

```bash
npm run check
```

## Project structure

```
corpus/
  sources.json              — source registry (metadata, copyright, display mode)
  sources/
    big-book-1ed.json       — Big Book 1st edition passages
    twelve-steps-traditions.json
  known-exceptions.json     — curated hints for common search terms
  scripts/
    build-index.mjs         — prebuilds the search index
    validate.js             — corpus schema validation

src/
  app.css                   — design tokens, global styles
  lib/
    types.ts                — core domain types
    corpus/
      registry.ts           — loads and validates sources.json
      exceptions.ts         — known-exception hint matcher
    search/
      index.ts              — MiniSearch hydration + search function
      kwic.ts               — KWIC clipping + highlight (XSS-safe)
    stores/
      online.ts             — online/offline reactive store
      toast.ts              — toast notification store
      version.ts            — index version check
    components/
      Nav.svelte            — sticky navigation bar
      ExternalLink.svelte   — offline-aware external link wrapper
      Toasts.svelte         — toast notification UI
  routes/
    +layout.svelte          — app shell (theme, nav, toasts)
    +page.svelte            — search / concordance (home)
    reflection/+page.svelte — Daily Reflection (links to aa.org)
    topics/+page.svelte     — topic browse
    sources/+page.svelte    — source registry display
    about/+page.svelte      — legal, privacy, open source
    passage/[sourceId]/[passageId]/+page.svelte — passage detail

static/
  icons/                    — PWA icons (192px, 512px)
  index/                    — generated by build:index (gitignored)
```

## Adding a new corpus source

1. Add a corpus file to `corpus/sources/<source-id>.json` per the schema in `corpus/CORPUS-GUIDE.md`
2. Add a registry entry to `corpus/sources.json`
3. Run `npm run build:index`
4. No application code changes required

See [corpus/CORPUS-GUIDE.md](corpus/CORPUS-GUIDE.md) for full instructions, copyright evaluation criteria, and ID conventions.

## Deploying to Cloudflare Pages

The app targets Cloudflare Pages with the `@sveltejs/adapter-cloudflare` adapter.

1. Push to the GitHub repo
2. Connect the repo to Cloudflare Pages:
   - Build command: `npm run build`
   - Build output directory: `.svelte-kit/cloudflare`
3. Add a custom domain in Pages → Custom domains

See the [Implementation Plan](docs/plans/basic-texts-implementation-plan.md) §0 for the full manual setup checklist.
